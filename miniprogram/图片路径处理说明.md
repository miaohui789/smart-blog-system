# 图片路径处理说明

## 问题描述

小程序中无法直接加载相对路径的图片，例如：
```
/uploads/2026/01/17/xxx.png  ❌ 无法加载
```

需要转换为完整的 URL：
```
http://localhost:8080/uploads/2026/01/17/xxx.png  ✅ 可以加载
```

## 解决方案

已创建 `utils/image.js` 工具类，自动处理图片路径转换。

### 使用方法

#### 1. 处理单个文章对象

```javascript
const imageUtil = require('../../utils/image')

// 加载文章后处理图片路径
let article = await articleApi.getDetail(id)
article = imageUtil.processArticleImages(article)

this.setData({ article })
```

#### 2. 处理文章列表

```javascript
const imageUtil = require('../../utils/image')

// 加载文章列表后处理图片路径
let articles = await articleApi.getList()
articles = imageUtil.processArticleList(articles)

this.setData({ articles })
```

#### 3. 处理用户对象

```javascript
const imageUtil = require('../../utils/image')

// 加载用户信息后处理图片路径
let userInfo = await userApi.getUserInfo()
userInfo = imageUtil.processUserImages(userInfo)

this.setData({ userInfo })
```

#### 4. 处理评论列表

```javascript
const imageUtil = require('../../utils/image')

// 加载评论列表后处理图片路径
let comments = await commentApi.getList(articleId)
comments = imageUtil.processCommentList(comments)

this.setData({ comments })
```

## 工具函数说明

### getImageUrl(path)
将相对路径转换为完整 URL

```javascript
getImageUrl('/uploads/xxx.png')
// 返回: http://localhost:8080/uploads/xxx.png
```

### getAvatarUrl(avatar)
获取用户头像完整 URL，如果为空返回默认头像

```javascript
getAvatarUrl('/uploads/avatar.png')
// 返回: http://localhost:8080/uploads/avatar.png

getAvatarUrl(null)
// 返回: 默认头像 URL
```

### getCoverUrl(cover)
获取文章封面完整 URL

```javascript
getCoverUrl('/uploads/cover.png')
// 返回: http://localhost:8080/uploads/cover.png
```

### processArticleImages(article)
处理文章对象中的所有图片路径

```javascript
const article = {
  cover: '/uploads/cover.png',
  author: {
    avatar: '/uploads/avatar.png'
  }
}

const processed = processArticleImages(article)
// 返回:
// {
//   cover: 'http://localhost:8080/uploads/cover.png',
//   author: {
//     avatar: 'http://localhost:8080/uploads/avatar.png'
//   }
// }
```

## 注意事项

1. **自动识别完整 URL**
   - 如果图片路径已经是 `http://` 或 `https://` 开头，不会重复处理

2. **默认头像**
   - 用户头像为空时，自动使用默认头像
   - 默认头像配置在 `utils/constants.js` 中

3. **环境自动切换**
   - 开发环境：自动使用 `http://localhost:8080`
   - 生产环境：自动使用配置的生产域名

4. **新页面开发**
   - 开发新页面时，记得引入 `imageUtil` 并处理图片路径
   - 参考已有页面的实现方式

## 示例代码

### 完整示例：文章列表页

```javascript
// pages/article/list/index.js
const articleApi = require('../../api/article')
const imageUtil = require('../../utils/image')

Page({
  data: {
    articles: []
  },

  async loadArticles() {
    try {
      // 1. 请求数据
      const res = await articleApi.getList()
      
      // 2. 处理图片路径
      const articles = imageUtil.processArticleList(res.records)
      
      // 3. 更新数据
      this.setData({ articles })
    } catch (err) {
      console.error('加载失败:', err)
    }
  }
})
```

## 常见问题

### Q: 图片还是加载失败？
A: 检查以下几点：
1. 确认已关闭域名校验
2. 确认后端服务正常运行
3. 确认图片文件确实存在
4. 查看控制台完整的图片 URL 是否正确

### Q: 如何修改默认头像？
A: 编辑 `utils/constants.js`：
```javascript
const DEFAULT_AVATAR = 'https://your-cdn.com/default-avatar.png'
```

### Q: 生产环境图片路径不对？
A: 修改 `utils/env.js` 中的生产环境配置：
```javascript
const prodConfig = {
  BASE_URL: 'https://your-domain.com/api'
}
```
