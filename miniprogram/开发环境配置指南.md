# 开发环境配置指南

## 🚨 域名校验错误解决方案

如果看到以下错误：
```
http://localhost:8080 不在以下 request 合法域名列表中
```

这是正常的，因为小程序默认只允许访问已配置的合法域名。

---

## ✅ 方法一：关闭域名校验（推荐）

### 步骤：

1. **打开微信开发者工具**

2. **点击右上角的"详情"按钮**

3. **切换到"本地设置"标签页**

4. **勾选以下选项：**
   ```
   ☑️ 不校验合法域名、web-view（业务域名）、TLS 版本以及 HTTPS 证书
   ```

5. **重新编译项目**

### 截图位置：
```
工具栏 → 详情 → 本地设置 → 不校验合法域名...
```

---

## 🔧 方法二：配置合法域名（生产环境）

### 前提条件：
- 已有备案的域名
- 后端服务已部署到 HTTPS 服务器

### 步骤：

1. **登录小程序后台**
   - 访问：https://mp.weixin.qq.com/

2. **进入开发设置**
   ```
   开发 → 开发管理 → 开发设置 → 服务器域名
   ```

3. **配置域名**
   - **request 合法域名**：`https://your-domain.com`
   - **socket 合法域名**：`wss://your-domain.com`
   - **uploadFile 合法域名**：`https://your-domain.com`
   - **downloadFile 合法域名**：`https://your-domain.com`

4. **修改配置文件**
   
   打开 `utils/env.js`，修改生产环境配置：
   ```javascript
   const prodConfig = {
     BASE_URL: 'https://your-domain.com/api',
     WS_URL: 'wss://your-domain.com/ws'
   }
   ```

## 🔍 常见问题

### Q1: 关闭域名校验后还是报错？
**A:** 
1. 确认已勾选"不校验合法域名"选项
2. 点击"清缓存" → "清除全部缓存"
3. 重新编译项目

### Q2: 真机调试时无法访问？
**A:** 
真机调试必须使用 HTTPS 域名，无法使用 localhost。
解决方案：
1. 使用内网穿透工具（如 ngrok、frp）
2. 部署到测试服务器

### Q3: 如何使用内网穿透？
**A:** 
使用 ngrok 示例：
```bash
# 安装 ngrok
npm install -g ngrok

# 启动内网穿透
ngrok http 8080

# 复制生成的 https 地址到 env.js
```

### Q4: 上传代码时提示域名未配置？
**A:** 
上传代码前必须：
1. 在小程序后台配置合法域名
2. 修改 `env.js` 中的生产环境配置
3. 确保后端已部署到 HTTPS 服务器
