# 博客网站项目开发文档

## 一、项目概述

### 1.1 项目简介
本项目是一个现代化的博客网站系统，采用前后端分离架构，包含用户端和管理员端两个子系统。
- **用户端**：提供博客浏览、评论、点赞、收藏等功能
- **管理员端**：提供内容管理、用户管理、数据统计等功能

### 1.2 设计风格
- **主题色**：紫色渐变 (#a855f7 → #ec4899)
- **背景色**：深色系 (#0f0f0f, #1a1a1a, #262626)
- **文字色**：白色/灰色 (#ffffff, #a1a1aa, #71717a)
- **卡片背景**：半透明深色 (#1f1f1f, rgba(255,255,255,0.05))
- **边框色**：深灰色 (#333333)

### 1.3 项目特点
- 响应式设计，支持PC端和移动端
- 深色主题，护眼舒适
- 紫色渐变点缀，现代感强
- 支持Markdown编辑器
- 完善的权限管理系统

## 七、Redis缓存设计

### 7.1 缓存Key设计

| Key格式 | 说明 | 过期时间 |
| --------------------------------- | -------------- | -------- |
| `blog:token:{userId}` | 用户Token | 7天 |
| `blog:user:{userId}` | 用户信息缓存 | 1小时 |
| `blog:article:{articleId}` | 文章详情缓存 | 30分钟 |
| `blog:article:view:{articleId}` | 文章浏览量 | 永久 |
| `blog:article:like:{articleId}` | 文章点赞用户集合 | 永久 |
| `blog:article:hot` | 热门文章列表 | 1小时 |
| `blog:category:list` | 分类列表缓存 | 1小时 |
| `blog:tag:list` | 标签列表缓存 | 1小时 |
| `blog:config` | 系统配置缓存 | 永久 |
| `blog:rate:{ip}:{uri}` | 接口限流 | 1分钟 |
| `blog:captcha:{uuid}` | 验证码 | 5分钟 |

### 7.2 缓存策略

1. **文章浏览量**: 使用Redis的INCR命令实时统计，定时同步到MySQL
2. **文章点赞**: 使用Redis的Set存储点赞用户ID，支持快速判断是否点赞
3. **热门文章**: 使用Redis的ZSet按浏览量排序，定时更新
4. **接口限流**: 使用Redis的计数器实现滑动窗口限流

## 八、安全设计

### 8.1 认证流程

```
1. 用户登录 -> 验证用户名密码 -> 生成JWT Token
2. 前端存储Token到localStorage
3. 请求时在Header中携带Token
4. 后端拦截器验证Token有效性
5. Token过期后需重新登录
```

### 8.2 权限控制

- **RBAC模型**: 用户-角色-权限三级控制
- **菜单权限**: 控制前端菜单显示
- **按钮权限**: 控制页面按钮显示
- **接口权限**: 后端接口权限校验

### 8.3 安全措施

| 措施 | 说明 |
| ------------ | -------------------------------- |
| 密码加密 | BCrypt加密存储 |
| SQL注入防护 | MyBatis参数化查询 |
| XSS防护 | 输入内容过滤转义 |
| CSRF防护 | Token验证 |
| 接口限流 | Redis实现限流 |
| 敏感数据脱敏 | 手机号、邮箱等脱敏显示 |

## 十一、开发规范

### 11.1 前端开发规范

#### 命名规范
- 组件名: PascalCase (如 `ArticleCard.vue`)
- 文件名: kebab-case (如 `article-card.vue`) 或 PascalCase
- 变量名: camelCase
- 常量名: UPPER_SNAKE_CASE
- CSS类名: kebab-case

#### 目录规范
- 组件放在 `components/` 目录
- 页面放在 `views/` 目录
- API接口放在 `api/` 目录
- 工具函数放在 `utils/` 目录

#### 代码规范
- 使用 ESLint + Prettier 格式化代码
- 组件使用 `<script setup>` 语法
- 使用 TypeScript 类型注解（可选）

### 11.2 后端开发规范

#### 命名规范
- 类名: PascalCase
- 方法名: camelCase
- 常量名: UPPER_SNAKE_CASE
- 包名: 全小写

#### 分层规范
- Controller: 只做参数校验和结果封装
- Service: 业务逻辑处理
- Mapper: 数据访问

#### 注释规范
- 类注释: 说明类的作用
- 方法注释: 说明方法功能、参数、返回值
- 关键代码注释: 说明复杂逻辑

### 11.3 Git提交规范

```
feat: 新功能
fix: 修复bug
docs: 文档更新
style: 代码格式调整
refactor: 代码重构
perf: 性能优化
test: 测试相关
chore: 构建/工具相关
```

### 12.1 环境准备
