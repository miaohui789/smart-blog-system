# 学习模块企业版数据库说明

企业版 SQL 文件：[`docs/database/study_module_enterprise.sql`](E:/1111xiangmu/Smart-Blog-System/new/smart-blog-system/docs/database/study_module_enterprise.sql)

这版不是轻量演示表，而是按真实业务拆出的完整表结构，重点覆盖了这几类能力：

- 题库分类、标签、题目主数据
- 题目版本快照与导入批次追踪
- 用户学习进度、收藏、错题、笔记
- 抽查任务、任务明细、历史作答记录
- AI 打分、自评得分、最终得分、维度评分

## 表清单

1. `study_category`：学习分类表
2. `study_tag`：学习标签表
3. `study_import_batch`：题库导入批次表
4. `study_import_batch_detail`：题库导入批次明细表
5. `study_question`：学习题目主表
6. `study_question_tag`：题目标签关联表
7. `study_question_version`：题目版本表
8. `study_user_question_progress`：用户题目学习进度表
9. `study_user_question_note`：用户题目笔记表
10. `study_check_task`：抽查任务表
11. `study_check_task_item`：抽查任务明细表
12. `study_answer_record`：答题记录表
13. `study_ai_score_record`：AI评分记录表

## 为什么这一版更适合真实项目

### 1. 支持 AI 打分留痕

`study_ai_score_record` 单独记录：

- 使用的模型
- 提示词版本
- Token 消耗
- 维度得分
- 原始响应
- 错误信息

这样后面你无论是做“重评分”、成本统计，还是排查 AI 返回异常，都有数据依据。

### 2. 支持查看答案后再自评

`study_check_task_item` 和 `study_answer_record` 里都保留了：

- 是否查看过标准答案
- 查看答案时间
- 自评结果
- 自评分

这就能完整支撑你说的“支持查看答案自行评估得分”。

### 3. 支持题目修改后的历史回放

`study_question_version` 和 `study_check_task_item` 的题目快照字段可以确保：

- 今天题目被修改了
- 你依然能看到用户昨天抽查时面对的原题和原标准答案

这一点在企业项目里非常重要，否则统计会失真。

### 4. 支持批量导入和问题回溯

`study_import_batch` 和 `study_import_batch_detail` 可以处理：

- Markdown 批量导入
- 导入失败明细
- 重复题识别
- 原始内容追踪

后面你把 `面试题大全md` 导入数据库时就能直接用上。

## 执行方式

```bash
mysql -u root -p 你的数据库名 < docs/database/study_module_enterprise.sql
```

如果你后面要继续落后端代码，我建议按下面顺序开发：

1. 先做题库管理和 Markdown 导入
2. 再做学习页和学习进度
3. 再做抽查任务和答题提交
4. 最后接 AI 打分与评分结果展示
