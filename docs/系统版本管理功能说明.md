# 系统版本管理功能说明

## 功能概述

实现了完整的系统版本管理功能，包括管理员端的CRUD操作和用户端的版本历史展示。

## 快速安装

### 一键安装脚本

使用完整安装脚本快速部署版本管理功能：

```bash
# Linux/Mac
mysql -u root -p blog_db < docs/database/system_version_complete.sql

# Windows PowerShell
Get-Content "docs/database/system_version_complete.sql" -Encoding UTF8 | mysql -u root -p密码 blog_db
```

该脚本包含：
- ✅ 创建 `system_version` 表
- ✅ 配置管理端菜单（5个菜单项）
- ✅ 分配角色权限（管理员+编辑）
- ✅ 插入初始版本数据（1.0.0）

详细安装说明请查看：`docs/database/README_版本管理安装.md`

## 数据库表

### system_version 表结构

| 字段名 | 类型 | 说明 |
|--------|------|------|
| id | BIGINT | 版本ID（主键） |
| version_number | VARCHAR(50) | 版本号（如：1.0.0，唯一） |
| version_name | VARCHAR(100) | 版本名称 |
| release_date | DATE | 发布日期 |
| description | TEXT | 版本描述 |
| features | TEXT | 新增功能（JSON数组） |
| improvements | TEXT | 优化改进（JSON数组） |
| bug_fixes | TEXT | 修复问题（JSON数组） |
| is_major | TINYINT(1) | 是否为重大版本（0-否，1-是） |
| status | TINYINT(1) | 状态（0-隐藏，1-显示） |
| sort_order | INT | 排序（数字越大越靠前） |
| create_time | DATETIME | 创建时间 |
| update_time | DATETIME | 更新时间 |

## 使用说明

### 管理员操作流程

1. **登录管理员后台**
2. **进入系统管理 > 版本管理**
3. **点击"新增版本"按钮**
4. **填写版本信息**：
   - 版本号（必填，如：1.0.0）
   - 版本名称（必填，如：正式版上线）
   - 发布日期（必填）
   - 版本描述（选填）
   - 新增功能（可添加多条）
   - 优化改进（可添加多条）
   - 修复问题（可添加多条）
   - 是否重大版本（开关）
   - 显示状态（开关）
   - 排序（数字越大越靠前）
5. **点击"确定"保存**

### 用户端查看

1. **访问网站首页**
2. **点击导航栏"关于"**
3. **查看版本历史时间线**

### 角色权限

- **管理员角色（role_id=1）**：拥有所有权限（查看、新增、编辑、删除、更新状态）
- **编辑角色（role_id=2）**：拥有部分权限（查看、新增、编辑）

## 注意事项

1. **版本号唯一性**：系统会自动检查版本号是否重复
2. **权限控制**：只有管理员和编辑角色可以管理版本
3. **状态控制**：只有状态为"显示"的版本才会在用户端展示
4. **排序规则**：按排序值降序，再按发布日期降序
5. **重大版本**：重大版本会在用户端特殊标记和高亮显示
6. **菜单刷新**：执行SQL脚本后，需要退出登录重新登录，或清除浏览器缓存后刷新页面
