# 面试题导入脚本说明

脚本文件：[`scripts/import_study_questions.py`](E:/1111xiangmu/Smart-Blog-System/new/smart-blog-system/scripts/import_study_questions.py)

这个脚本用于把 [面试题大全md](E:/1111xiangmu/Smart-Blog-System/new/smart-blog-system/面试题大全md) 目录下的 Markdown 题库自动导入到学习模块企业版数据库表中。

## 支持能力

- 自动解析 `##` 章节和 `###` 题目
- 自动创建导入批次和导入明细
- 自动匹配或创建分类
- 自动新增题目
- 已存在题目内容变化时自动升级版本
- 内容没变化时自动跳过，避免重复写库
- 支持 `dry-run`
- 支持导入前自动执行企业版建表 SQL
- 优先使用 `PyMySQL`，未安装时自动回退到本机 `mysql` 客户端

## 默认配置来源

脚本默认会读取项目配置文件：

- [`application-dev.yml`](E:/1111xiangmu/Smart-Blog-System/new/smart-blog-system/backend/src/main/resources/application-dev.yml)

默认识别到的数据库配置是：

- host: `localhost`
- port: `3306`
- database: `blog_db`
- username: `root`
- password: `123456`

## 常用命令

### 1. 只解析，不入库

```bash
python scripts/import_study_questions.py --dry-run
```

### 2. 自动建表后再导入

```bash
python scripts/import_study_questions.py --bootstrap
```

### 3. 只导入一个文件

```bash
python scripts/import_study_questions.py --only-file "Java八股文200道-Java基础篇.md"
```

### 4. 只测试前 10 道题

```bash
python scripts/import_study_questions.py --limit 10
```

### 5. 指定数据库连接

```bash
python scripts/import_study_questions.py --database blog_db --username root --password 123456
```

### 6. 强制使用 mysql 客户端

```bash
python scripts/import_study_questions.py --force-cli
```

## 导入目标表

脚本会写入这些表：

1. `study_category`
2. `study_import_batch`
3. `study_import_batch_detail`
4. `study_question`
5. `study_question_version`

## 注意事项

### 1. 建议先执行企业版建表 SQL

如果你还没建表，可以直接使用：

```bash
python scripts/import_study_questions.py --bootstrap
```

或者先手动执行：

```bash
mysql -u root -p 你的数据库名 < docs/database/study_module_enterprise.sql
```

### 2. 题目判重规则

脚本优先按：

- `category_id + question_no`

进行匹配。

如果题号缺失，则退化为：

- `category_id + title`

### 3. 版本升级规则

如果以下字段发生变化，会自动给题目升一个新版本并写入 `study_question_version`：

- `title`
- `standard_answer`
- `answer_summary`
- `difficulty`

### 4. 当前脚本不做的事情

这版脚本先聚焦题库主数据导入，暂时不自动生成：

- `study_tag`
- `study_question_tag`
- 学习进度数据
- 抽查任务数据
- AI评分数据

这些是运行期数据，不适合在题库导入阶段直接生成。
