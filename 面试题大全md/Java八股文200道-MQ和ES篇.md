## 十一、MQ

### 141、什么是MQ？为什么用MQ？

```
MQ 是 Message Queue（消息队列）的简称，是一种存储和转发消息的中间件，核心是解耦消息发送方和接收方。

核心作用
解耦：发送方只需发消息到 MQ，无需关心谁接收，下游系统增减不影响上游。
削峰：应对秒杀等突发流量，先存消息再让下游匀速处理，避免系统崩溃。
异步：发送方发完消息即返回，无需等待接收方处理，提升响应速度。
```



### 142、常见的MQ产品

```
RabbitMQ：订单同步、延时消息等需复杂机制的中小流量场景。
Kafka：日志采集、实时数据分析等高吞吐大数据场景。
MQTT：传感器上报、APP 推送等 IoT 或低带宽场景。
```



### 143、**如何保证消息的可靠传输？**

```
保证消息可靠传输，抓三个环节：
生产端：开发送确认（如 RabbitMQ 的 confirm、Kafka 的 acks=all），失败重试，关键消息本地落库兜底。
MQ 端：消息持久化到磁盘，集群多副本（如 Kafka 副本、RabbitMQ 镜像队列），防止宕机丢失。
消费端：手动确认（处理完再 ack），用唯一 ID 保证幂等（防重复），失败消息入死信队列。
```



### 144、RabbitMQ工作模式？

```
RabbitMQ 有 7 种核心工作模式，简洁要说明：

简单模式：1 个生产者 → 1 个队列 → 1 个消费者，直接点对点传递。
工作队列模式：1 个生产者 → 1 个队列 → 多个消费者，任务分发（默认轮询，可改公平分发）。
发布/订阅模式：1 个生产者 → 1 个交换机（fanout 类型）→ 多个队列 → 多个消费者，广播消息（每个消费者都能收到）。
路由模式：交换机为 direct 类型，消息带 routing key，队列绑定指定 key，只有匹配的消费者能收到。
主题模式：交换机为 topic 类型，routing key 支持通配符（* 匹配 1 个词，# 匹配多个），更灵活的路由。
RPC 模式：客户端发请求消息，服务端处理后返回响应，模拟远程调用。
发布/确认模式：保证生产者消息可靠发送到 MQ 的核心机制，确保消息不丢失。
```



### 145、RabbitMQ**交换机类型及区别？**

```
 fanout：广播，忽略路由键；
 direct：精确匹配路由键；
 topic：通配符匹配（* 单词，# 多词）；
 headers：按消息头匹配（少用）。
```



### 146、**如何避免消息重复消费？**

```
唯一标识 + 幂等校验：
消息带唯一 ID（如 UUID），消费时先查数据库 / 缓存是否处理过。
已处理则直接返回，未处理则执行业务 + 记录 ID。
```



### 147、什么是死信队列？

```
死信队列（Dead-Letter Queue，简称 DLQ）是 RabbitMQ 中专门存放 “无法正常消费” 消息的队列，避免这些消息阻塞正常队列或丢失，方便后续排查处理。
一、消息成为 “死信” 的 3 种情况
消息过期：消息设置了 TTL（存活时间），到期后仍未被消费。
队列满了：目标队列达到最大容量，新消息无法入队。
消费被拒：消费者明确拒绝消息（调用basicNack/basicReject），且未设置 “重回原队列”（requeue=false）。
二、核心作用
隔离异常消息：将死信与正常消息分开，不影响正常业务消费。
便于问题排查：后续可分析死信原因（如业务逻辑错误、数据异常），甚至手动重试消费。
三、典型场景
比如 “订单超时未支付”：订单消息设置 30 分钟 TTL，若 30 分钟内未支付，消息成为死信进入 DLQ，后续可从 DLQ 读取消息，执行 “取消订单、释放库存” 的操作。
```



## 十二、ES

### 148、什么是es？作用是什么？

```
ES 是 Elasticsearch 的简称，是基于 Lucene 的分布式全文搜索引擎，核心用于海量数据的快速检索与分析。

核心作用
全文检索：分词匹配关键词（如电商商品搜索）。
数据聚合：多维度统计分析（如按地区统计订单）。
近实时：数据写入后秒级可查（如日志监控）。
分布式扩展：支持 TB/PB 级数据存储，可横向扩容。
```



### 149、倒排索引

```
倒排索引（Inverted Index）是 ES 等搜索引擎的核心数据结构，用于快速定位包含特定关键词的文档。

核心结构：
词项（Term）：文本分词后的关键词（如 “苹果手机” 拆分为 “苹果”“手机”）。
文档列表（Postings List）：包含该词项的所有文档 ID，还可记录位置、频率等信息。

作用：
让搜索引擎从 “遍历所有文档找关键词” 变为 “直接查关键词对应的文档”，大幅提升检索效率（毫秒级响应）。
```



### 150、什么是ELK？

```
ELK 是三个开源工具的组合缩写：
E（Elasticsearch）：分布式搜索引擎，负责存储、检索和分析数据。
L（Logstash）：数据采集与处理工具，可收集、过滤、转换日志等数据。
K（Kibana）：可视化平台，通过图表展示 ES 中的数据（如日志报表、监控仪表盘）。

三者配合形成日志收集→处理→存储→分析→可视化的完整链路，常用于日志监控、数据分析等场景。
```
